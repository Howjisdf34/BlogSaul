DQL: El lenguaje para consultar información en bases de datos

Aprende qué es DQL, para qué sirve y cómo se usa dentro de SQL para obtener información almacenada en una base de datos.

Comenzar lectura

¿Qué es DQL?

DQL significa Data Query Language, que en español se traduce como Lenguaje de Consulta de Datos. Es una parte de SQL que se utiliza para consultar, buscar y recuperar información guardada dentro de una base de datos.

A diferencia de otros lenguajes de SQL, DQL no se usa para crear tablas, modificar estructuras o eliminar datos. Su función principal es mostrar información de acuerdo con las condiciones que indique el usuario.

Idea clave: DQL sirve para hacer preguntas a la base de datos y recibir respuestas en forma de tablas o resultados.

Importancia de DQL en bases de datos

En cualquier sistema que maneje información, como una tienda, una escuela, un banco o una red social, es necesario consultar datos constantemente. Por ejemplo, se puede querer saber qué clientes compraron un producto, cuántos alumnos aprobaron una materia o cuáles empleados pertenecen a cierto departamento.

DQL permite obtener esa información de forma rápida, ordenada y precisa. Por eso es una herramienta esencial para programadores, administradores de bases de datos y analistas de datos.

Comandos principales de DQL

El comando más importante dentro de DQL es SELECT.

SELECT

Indica qué columnas o datos se desean consultar.

FROM

Indica de qué tabla se obtendrá la información.

WHERE

Permite filtrar los resultados usando condiciones.

ORDER BY

Ordena los resultados de forma ascendente o descendente.

GROUP BY

Agrupa registros que tienen valores similares.

HAVING

Filtra grupos después de usar GROUP BY.

Ejemplos básicos de consultas DQL

1. Consultar todos los datos de una tabla

Este ejemplo muestra todos los registros de la tabla clientes.

SELECT * FROM clientes;

2. Consultar columnas específicas

Este ejemplo solo muestra el nombre y teléfono de los clientes.

SELECT nombre, telefono FROM clientes;

3. Usar una condición con WHERE

Este ejemplo muestra los productos cuyo precio sea mayor a 100.

SELECT nombre_producto, precio
FROM productos
WHERE precio > 100;

4. Ordenar resultados

Este ejemplo ordena los productos desde el más caro hasta el más barato.

SELECT nombre_producto, precio
FROM productos
ORDER BY precio DESC;

5. Agrupar datos

Este ejemplo cuenta cuántos productos existen por categoría.

SELECT categoria, COUNT(*) AS total_productos
FROM productos
GROUP BY categoria;

DQL y las funciones de agregación

Las funciones de agregación permiten realizar cálculos sobre los datos consultados. Normalmente se usan junto con GROUP BY.

Función Uso
COUNT() Cuenta registros.
SUM() Suma valores numéricos.
AVG() Calcula el promedio.
MAX() Obtiene el valor máximo.
MIN() Obtiene el valor mínimo.

DQL con JOIN

En bases de datos relacionales, muchas veces la información se encuentra dividida en varias tablas. Para unir esos datos se utiliza JOIN.

Ejemplo: mostrar el nombre del cliente y el producto que compró.

SELECT clientes.nombre, productos.nombre_producto
FROM ventas
JOIN clientes ON ventas.id_cliente = clientes.id_cliente
JOIN productos ON ventas.id_producto = productos.id_producto;

Buenas prácticas al usar DQL

  • Evita usar SELECT * cuando solo necesitas algunas columnas.
  • Usa nombres claros para tablas y columnas.
  • Filtra los resultados con WHERE para no cargar datos innecesarios.
  • Ordena los resultados solo cuando sea necesario.
  • Usa alias para que tus consultas sean más fáciles de leer.

Conclusión

DQL es una parte fundamental de SQL porque permite consultar información dentro de una base de datos. Su comando principal es SELECT, el cual puede combinarse con otras instrucciones como WHERE, ORDER BY, GROUP BY, HAVING y JOIN.

Dominar DQL ayuda a obtener datos de manera eficiente y a comprender mejor cómo se organiza la información en los sistemas.
