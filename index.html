Comprendiendo DDL y DML

La forma más sencilla de entender cómo se define y cómo se manipula una base de datos en SQL, sin rodeos.

La regla de oro 💡

Piensa en una base de datos como una **casa**. El lenguaje DDL define los planos y construye las paredes de la casa. El lenguaje DML es el que te permite meter, cambiar o sacar los muebles de adentro.

Definición de Estructura

¿Qué es DDL?

Data Definition Language se usa para crear, modificar y destruir la estructura de los objetos de la base de datos (tablas, índices, etc.). No toca los registros.

Comandos principales:

  • CREATE Crea tablas o bases de datos nuevas.
  • ALTER Modifica la estructura (añade/borra columnas).
  • DROP Elimina tablas u objetos por completo.
Manipulación de Datos

¿Qué es DML?

Data Manipulation Language te permite trabajar con los datos reales que viven dentro de las tablas. Se encarga de las consultas y modificaciones de registros.

Comandos principales:

  • SELECT Consulta o recupera registros específicos.
  • INSERT Agrega nuevas filas de datos a una tabla.
  • UPDATE Modifica datos que ya existen.
  • DELETE Elimina registros de la tabla.

Resumen de un vistazo

Criterio DDL DML
¿Qué modifica? El esquema / estructura (tablas, columnas). Los registros de datos (filas).
¿Se puede deshacer? No fácilmente (Auto-commit). Sí, permite transacciones (Rollback/Commit).
Comando ejemplo ALTER TABLE usuarios ADD correo VARCHAR; INSERT INTO usuarios VALUES ('Juan');
